## 基础过滤规则

### 1.1 固定FOV过滤 (FOVBoundaryFilter)

#### 个工作原理

过滤器将3D笛卡尔坐标转换为相对摄像头的角度坐标，然后与FOV范围对比。

**坐标转换公式：**

```
给定点 P = (x, y, z) 在摄像头坐标系中
摄像头指向 +z 方向（深度方向）

水平角 (左右偏离):
  θ_h = arctan2(x, z)
  范围: [-π, π]   或   [-180°, 180°]
  
竖直角 (上下偏离):
  θ_v = arctan2(y, z)
  范围: [-π, π]   或   [-180°, 180°]
```

#### 过滤判断条件

点被**保留**的条件（两个条件都要满足）：

```
保留 ⟺ (|θ_h| ≤ FOV_h/2) AND (|θ_v| ≤ FOV_v/2)

过滤 ⟺ (|θ_h| > FOV_h/2) OR (|θ_v| > FOV_v/2)
```

#### OAK-D默认参数

```python
FOV_h = 72°   （水平视野，左右各36°）
FOV_v = 53°   （竖直视野，上下各26.5°）
margin = 0°   （可选安全裕度）

有效过滤范围：
  水平: -36° 到 +36°
  竖直: -26.5° 到 +26.5°
```

#### 视觉示意图

**俯视图（从上往下看）：**
```
        北
        ↑
        │ FOV边界
        │
   西 ←-O-→ 东 (摄像头)
        │
        │ 南
        
      θ_h角度分布

    -36°┃+36°
       ╱│╲
      ╱ │ ╲  有效FOV区域
     │  │  │
  ───┼──O──┼───
     │  │  │
      ╲ │ ╱
       ╲│╱
```

**侧视图（从右往左看）：**
```
        上
        ↑
     +26.5°
        │
   ────O───── 摄像头 → z轴(深度)
        │
     -26.5°
        ↓
        下
```

### 1.2 边界裕度机制 (margin)

#### 作用
为边界增加保护区域，防止边界附近的"异常点"被错误保留。

#### 计算方式

```python
# 裕度的应用
有效FOV半角 = 原始FOV半角 - margin

例如：
  margin = 0°:   有效范围 = [-36°, +36°]  （严格边界）
  margin = 1°:   有效范围 = [-35°, +35°]  （宽容1°）
  margin = 2°:   有效范围 = [-34°, +34°]  （宽容2°）
```

#### 推荐配置

| 场景 | 距离 | 推荐margin | 理由 |
|------|------|-----------|------|
| 近距离精确提取 | <1m | 0-1° | 浮点误差小，需要精度 |
| 中距离平衡 | 1-3m | 1-2° | 折中精度和容错 |
| 远距离稳定性 | >3m | 2-3° | 相机畸变更明显 |
| 低噪声环境 | 任意 | 0° | 源数据质量好 |
| 高噪声环境 | 任意 | 2-3° | 防止噪声被保留 |

## 自适应过滤规则

### 2.1 整体工作流程

```
输入: 原始点云 P = {p_1, p_2, ..., p_N}
      │
      ├─ Step 1: 分析点云特性
      │   ├─ 计算边界密度 (BoundaryDensity)
      │   ├─ 计算深度方差 (DepthVariance)
      │   └─ 检测离群点比例 (OutlierRatio)
      │
      ├─ Step 2: 计算自适应评分
      │   └─ score = weighted_sum(BD, DV, OR)
      │
      ├─ Step 3: 生成自适应margin
      │   └─ margin_adapt = margin_base + 调整量
      │
      ├─ Step 4: 应用新margin重新过滤
      │   └─ 使用 θ_h, θ_v 与新 margin 比较
      │
      └─ 输出: 过滤后的点云 + 统计信息
```

### 2.2 三个自适应检测指标

#### ① 边界密度自适应 (BoundaryDensity)

**计算方法：**
```python
# 检查距离FOV边界 0-5° 范围内的点
boundary_margin_range = 5°

# 识别边界附近的点
near_boundary = points where:
    (|θ_h| > FOV_h/2 - boundary_margin_range) AND (|θ_h| ≤ FOV_h/2)
    OR
    (|θ_v| > FOV_v/2 - boundary_margin_range) AND (|θ_v| ≤ FOV_v/2)

# 计算密度
BoundaryDensity = count(near_boundary) / total_count
范围: [0.0, 1.0]
```

**场景解释：**
| BD值 | 含义 | 相机状态 | 调整动作 |
|-----|------|---------|---------|
| 0.0-0.1 | 很少点在边界 | 正常 | 不调整 |
| 0.1-0.3 | 部分点在边界 | 轻微异常 | 增加 0-0.9° |
| 0.3-0.6 | 大量点在边界 | 明显异常 | 增加 0.9-1.8° |
| 0.6-1.0 | 几乎全在边界 | 严重异常 | 增加 1.8-3.0° |

**触发原因：**
- ✓ 相机镜头有明显畸变
- ✓ 目标物体全部位于边界附近
- ✓ 深度传感器在边界处置信度下降
- ✗ 通常情况下不触发 (BD < 0.15)

#### ② 深度方差自适应 (DepthVariance)

**计算方法：**
```python
# 使用变异系数 (Coefficient of Variation)
z_values = [p[2] for p in points]  # 所有点的z坐标（深度）

mean_z = mean(z_values)
std_z = std(z_values)

DepthVariance = std_z / mean_z    # 变异系数
范围: [0.0, ∞)，通常 [0.0, 1.0]
```

**场景解释：**
| DV值 | 含义 | 场景 | 调整动作 |
|-----|------|------|---------|
| 0.0-0.2 | 深度很均匀 | 单一物体 | 不调整 |
| 0.2-0.4 | 深度相对均匀 | 近距物体 | 增加 0-0.8° |
| 0.4-0.6 | 深度差异显著 | 复杂场景 | 增加 0.8-1.2° |
| >0.6 | 深度分布混乱 | 高度非结构 | 增加 1.2-2.0° |

**解释原理：**
```
DepthVariance 反映场景复杂程度

简单场景 (DV=0.1):        复杂场景 (DV=0.7):
  |近物体                    |近物体
  |                          |\
  |                          | \远物体
  └──────────→              └──────────→
     深度                        深度

复杂场景中的边界点因深度测估不准
而容易产生"飘浮"现象，需要保守过滤
```

**触发原因：**
- ✓ 场景包含多层物体（桌上有杯子）
- ✓ 深度跳跃（前景-背景）
- ✓ 相机到物体距离变化大
- ✗ 平面场景 (DV < 0.2)

#### ③ 离群点检测自适应 (OutlierRatio)

**计算方法：**
```python
# 计算每个点到摄像头的距离
distances = [||p|| for p in points]  # 欧氏距离

mean_dist = mean(distances)
std_dist = std(distances)

# 识别离群点 (3-sigma 规则)
SIGMA_THRESHOLD = 3
outliers = points where:
    |distance - mean_dist| > SIGMA_THRESHOLD × std_dist

OutlierRatio = count(outliers) / total_count
范围: [0.0, 1.0]
```

**场景解释：**
| OR值 | 含义 | 问题类型 | 调整动作 |
|-----|------|---------|---------|
| 0.0-0.05 | 极少离群点 | 高质量数据 | 不调整 |
| 0.05-0.15 | 少量离群点 | 轻微噪声 | 增加 0-0.75° |
| 0.15-0.3 | 中量离群点 | 镜面反射 | 增加 0.75-1.35° |
| >0.3 | 严重离群 | 测量失败 | 增加 1.35-1.5° |

**离群点来源：**
```
1. 镜面反射 → 虚假深度点
   [实物]
      ↓
   [镜子]← 反射光线混乱
      ↓
   [错误测量] → "飘浮"在远处

2. 深度传感器噪声
   - 干扰光线
   - 饱和区域
   - 低对比区域

3. 多路径错误
   - 同一光线多次反射
   - 多个物体共同贡献
```

**触发原因：**
- ✓ 环境有镜子或玻璃
- ✓ 相机曝光设置不当
- ✓ 高反光表面
- ✓ 低光或高光对比
- ✗ 正常室内场景 (OR < 0.10)

### 2.3 自适应融合公式

#### 权重配置
```python
# 默认配置（均衡）
w_density = 0.5   # 边界密度权重
w_depth = 0.3     # 深度方差权重
w_outlier = 0.2   # 离群点权重（自动计算）

验证: 0.5 + 0.3 + 0.2 = 1.0 ✓
```

#### 调整量计算
```python
# 同时满足三个调整
adjustment = (BD × 3.0 × w_density) +
             (DV × 2.0 × w_depth) +
             (OR × 1.5 × w_outlier)

其中：
  BD × 3.0    ← BD最多产生3°调整
  DV × 2.0    ← DV最多产生2°调整
  OR × 1.5    ← OR最多产生1.5°调整
```

#### 最终margin计算
```python
adaptive_margin = base_margin + adjustment

# 安全限制
adaptive_margin = clamp(adaptive_margin, min=0.1°, max=10.0°)

// 防止：
// - 过小: < 0.1° 会导致有效过滤范围太大
// - 过大: > 10° 会过度过滤，丢失有效点
```

#### 具体计算示例

**场景：复杂室内环境**
```
输入点云特性：
  - BD = 0.35  (35%的点在边界)
  - DV = 0.52  (52%的深度变异系数)
  - OR = 0.12  (12%的离群点)
  - base_margin = 2.0°

计算步骤：
  BD_adjust = 0.35 × 3.0 × 0.5 = 0.525°
  DV_adjust = 0.52 × 2.0 × 0.3 = 0.312°
  OR_adjust = 0.12 × 1.5 × 0.2 = 0.036°
  
  total_adjust = 0.525 + 0.312 + 0.036 = 0.873°
  
  adaptive_margin = 2.0 + 0.873 = 2.873°
  final_margin = clamp(2.873, 0.1, 10.0) = 2.873°

过滤范围调整：
  原始: ±36° (FOV_h/2 - 2.0°) = ±34°
  自适应: ±36° (FOV_h/2 - 2.873°) = ±33.127°
  
  结果：多过滤了约 0.873° 范围内的点
```

### 2.4 权重调整策略

#### 场景1: 最大化点数覆盖
```python
w_density = 0.3    # 减少边界过滤
w_depth = 0.2      # 轻松深度方差检查
w_outlier = 0.5    # 严格离群点检查

效果: 保留尽可能多的点，但去除明显异常

适用:
  - 建图任务(SLAM)
  - 点云采集(最大覆盖)
  - 低精度应用
```

#### 场景2: 最高精度提取
```python
w_density = 0.6    # 积极过滤边界
w_depth = 0.3      # 保守对待深度
w_outlier = 0.1    # 信任大部分点

效果: 只保留最可信的点，丢弃边界可疑点

适用:
  - 精确物体检测
  - 点云配准
  - 高精度测量
```

#### 场景3: 平衡配置（推荐）
```python
w_density = 0.5    # 标准边界过滤
w_depth = 0.3      # 标准深度检查
w_outlier = 0.2    # 标准离群检查

效果: 在精度和覆盖间取得平衡

适用:
  - 大多数实际应用
  - UAV避障
  - 3D重建
```

## 过滤决策矩阵

### 3.1 固定过滤决策树

```
START: 输入点 P = (x, y, z)
  │
  ├─[Step 1] 计算角度
  │  θ_h = atan2(x, z)
  │  θ_v = atan2(y, z)
  │
  ├─[Step 2] 检查水平角
  │  │
  │  ├─ |θ_h| ≤ 36°? ──→ NO ──→ [删除] ✗
  │  └─ YES 继续
  │
  ├─[Step 3] 检查竖直角
  │  │
  │  ├─ |θ_v| ≤ 26.5°? ──→ NO ──→ [删除] ✗
  │  └─ YES 继续
  │
  └─[END] [保留] ✓
     返回点P
```

### 3.2 自适应过滤决策树

```
START: 输入点云 P
  │
  ├─[分析阶段]
  │  ├─ 计算 BD (边界密度)
  │  ├─ 计算 DV (深度方差)
  │  ├─ 计算 OR (离群比例)
  │  └─ 融合得到 margin_adapt
  │
  ├─[对每个点做固定过滤]
  │  ├─ 使用 margin_adapt 而非 margin_base
  │  └─ 参考 "固定过滤决策树"
  │
  └─[END] 返回过滤后点云 + 统计信息

示例：
  如果 BD 高 → margin⬆ → FOV范围⬇ → 点保留⬇
  如果 OR 高 → margin⬆ → FOV范围⬇ → 点保留⬇
```

### 3.3 边界分类

```
0°: 中心点（最可信）
    ↓
15°: 近边界点（可信）
    ↓
30°: 很近边界（可疑）
    ↓
35°: 极近边界（很可疑）
    ↓
36°+: 完全外边界（过滤）
```

## 性能特性

### 4.1 时间复杂度

| 操作 | 复杂度 | 说明 |
|------|--------|------|

| 掩膜生成 | O(H×W) | H×W=分辨率 |

## 使用建议

### 5.1 过滤方式选择

#### 使用固定FOV过滤的场景
```python
from oakd_perception.fov_boundary_filter import FOVBoundaryFilter

filter = FOVBoundaryFilter(fov_h=72.0, fov_v=53.0, margin=2.0)
filtered = filter.filter_fov_boundary(points)
```

**优点：**
- ✓ 速度快 (约固定时间)
- ✓ 参数固定，行为可预测
- ✓ 易于调试
- ✓ 适合实时系统

**适用场景：**
- 实时应用(避障、导航)
- 场景相对稳定
- 计算资源受限
- 需要最小延迟

#### 使用自适应FOV过滤的场景
```python
from oakd_perception.fov_boundary_filter import AdaptiveFOVBoundaryFilter

filter = AdaptiveFOVBoundaryFilter(base_fov_h=72.0, base_fov_v=53.0)
filtered, stats = filter.filter_adaptive(points, auto_adapt=True)
```

**优点：**
- ✓ 自动适应多变场景
- ✓ 无需手工调参
- ✓ 鲁棒性好
- ✓ 支持动态权重配置

**适用场景：**
- 场景多变
- 相机特性未知
- 可接受微小延迟(1-2ms)
- 需要最高鲁棒性

### 5.3 调试流程

1. **分析点云特性**
   ```python
   stats = get_fov_statistics(points)
   print(f"边界外点比例: {stats['boundary_ratio']:.2%}")
   ```

2. **尝试固定过滤**
   ```python
   filtered = remove_fov_boundary_points(points, margin=2.0)
   print(f"保留: {len(filtered)}/{len(points)}")
   ```

3. **评估结果**
   - 是否丢失了关键点?
   - 是否保留了边界噪声?

4. **调整参数/切换方式**
   ```python
   # 方案A: 增加margin
   filtered = remove_fov_boundary_points(points, margin=3.0)
   
   # 方案B: 切换自适应
   filtered, stats = adaptive_filter_fov_boundary(points)
   ```

## 实际案例

### 6.1 案例1: UAV室内避障

**场景描述：**
- 场景: 办公室，桌子、椅子、墙壁
- 摄像头: OAK-D，72°×53° FOV
- 任务: 实时避障

**点云特性分析：**
```
原始点数: 307,200
- 中心区域点: 280,000
- 边界区域点: 27,200 (8.8%)
- 超出FOV点: 2,000 (0.7%)

深度分布:
- 最近: 0.2m (桌面)
- 最远: 5m (远墙)
- 变异系数: 0.45

离群点:
- 镜面反射(玻璃): 1500点
- 饱和区域: 500点
- 比例: 0.64%
```

**推荐配置：**
```python
# 固定过滤 - 速度优先
filter = FOVBoundaryFilter(fov_h=72.0, fov_v=53.0, margin=1.5)
filtered = filter.filter_fov_boundary(points)

# 结果: 29,500点被过滤，保留277,700点 (90.4%)
# 运行时间: 5ms
# FPS: 200 FPS (远超30FPS要求)
```

## 总结表

| 特性 | 固定过滤 | 自适应过滤 |
|------|---------|----------|
| **速度** | 快(1×) | 中等(3×) |
| **精度** | 局限 | 很高 |
| **鲁棒性** | 差 | 很好 |
| **调参成本** | 高 | 低 |
| **计算延迟** | <1ms | 1-3ms |
| **最佳应用** | 实时系统 | 多场景应用 |
| **推荐场景** | 避障、导航 | 建图、采集、检测 |

---

**最后建议：** 推荐优先使用 **自适应过滤**，因为它自动处理大多数场景，无需手工调参。仅在对延迟要求极严格的实时系统中才切换到固定过滤。
